# Internationalization (i18n)

Rules for adding or changing translations.

## Where translations live

- Keep translations under each package's `src/assets/i18n/*.json`.
- Translations are merged from module-local files into bundled outputs at build time.
- **Avoid duplicate keys across modules for the same language** — the merge plugin throws on conflicts.

## Validating

After adding or changing translations, run:

```bash
npm run validate
```

If validation fails, inspect the generated `translation-report.json`.

## Runtime translations in the workbench

The Angular workbench renders translations via Transloco — see [frontend-angular.md](frontend-angular.md#translations-transloco).
